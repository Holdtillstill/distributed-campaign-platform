# Code of Conduct

This repository is intended to be a practical, respectful portfolio project for distributed systems and platform engineering work.

## Expected Behavior

- Be constructive and specific when discussing bugs, design choices, tests, operations, and documentation.
- Assume good intent, but keep feedback grounded in evidence and project goals.
- Keep issues and pull requests focused on the repository's public scope.
- Respect privacy and operational safety. Do not post secrets, cloud identifiers, private manifests, campaign recipient data, visitor data, or private preview URLs.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private or sensitive information.
- Repeated off-topic comments, spam, or attempts to bypass security and cost guardrails.
- Encouraging unsafe changes to real cloud or Kubernetes environments.

## Reporting

For conduct concerns, use the repository owner contact path. For security-sensitive concerns, follow [SECURITY.md](SECURITY.md) instead of opening a public issue.
